RealityWarden Desktop Support

Offline guide · Public Alpha · simulation first

Recover from startup

  1. Use Retry startup in the recovery window.
  2. If retry fails, use Copy details, then export diagnostics from File → Export Local Diagnostics….
  3. Restart RealityWarden. Uninstall/reinstall preserves user data by default.

An unavailable local LLM does not block startup. Supported commands fall back explicitly to the local rule compiler.

Recover a project

  • Open a .openreality.json file with File → Open Project.
  • Use File → Restore for the last validated autosave.
  • Corrupt autosaves are quarantined and never silently replace the current workspace.

Local diagnostics and privacy

The JSON bundle is created only after you choose a destination. RealityWarden does not upload it.

It contains runtime/version metadata plus a bounded, allowlisted, redacted startup-log excerpt. It excludes projects, imported assets, prompts, generated commands, audit evidence, hardware results, serial ports, environment variables, and credentials. Review it before sharing.

REAL HARDWARE boundary

REAL HARDWARE is a separate evidence-locked reference-rig path. It requires explicit operator confirmation. Missing, stale, invalid, or frozen sensor evidence blocks execution. A software acknowledgement is never presented as proof of physical motion.

Runnable Public Alpha paths

Robot Arm, Smart Light, and Camera Sensor are runnable in simulation. Other displayed device families are unfinished and cannot be used as release evidence.
