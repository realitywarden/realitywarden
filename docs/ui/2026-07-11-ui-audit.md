# UI 审查报告（2026-07-11）

**性质**：只审不改。基于真实运行截图（1920×1080，英文模式，Robot Arm 场景 EXECUTED 后）+ 前端代码通读（`app/page.tsx` 2814 行、`components/*` 全部、`lib/i18n.ts`）。
**分级**：P0 = 重叠/误导/信息错误，用户直接受害；P1 = 明显混乱但可绕过；P2 = 一致性与打磨。
**冻结令标注**：〔可修〕= 属修 bug 范畴，冻结期内可动手；〔重构〕= 属设计变更，真机验收后处理。

**总评**：单屏问题的根源不是零散 bug，而是三个系统性缺陷——**同一信息无单一权威展示位**（到处重复）、**视口覆盖层无布局管理**（绝对定位互不知情，必然重叠）、**状态语义无统一词表**（EXECUTED/COMPLETED/COMPILED/READY 四个词说三件事）。修零散 bug 只能止血；三个根因都需要在验收后做一次信息架构收敛。

## A. 重叠与裁切（P0，全部〔可修〕）

### A1. 视口右上"常驻假 tooltip"与中央状态条同线重叠
`VirtualDeviceStage.tsx:152`（中央条 `left-1/2 top-12`）与 `:164`（右上覆盖层 `right-3 top-12` `min-w-[210px] max-w-[280px]`）在同一水平线。中央条**无 max-width、无 truncate**，内容 = "Active Workspace Device -> {名} | Current Run Target -> {名} | Observe Here"，同一设备名拼两遍，长名（"Generic Industrial Robot Arm"）下总宽 ~700px+。视口宽 <~1500px 时两者必然相撞——截图正是此状态，右上层直接盖住 "Current R…"。
根上 `page.tsx:2582` 声明 `min-w-[1180px]`，即官方支持的最小宽度里有 ~320px 的必撞区间。
**修**：中央条限宽 + truncate + 设备名只出现一次；右上层上移到 `top-3` 行或与中央条合并；1180px 下逐层验证。

### A2. 右上覆盖层长得像卡死的 tooltip
`:164-175` 是常驻信息面板，但视觉是浮层（黑底模糊、小字、READY chip），用户第一反应是"悬停提示没消失"。**修**：要么改成真 hover（跟随选中设备），要么给它面板化外观（贴右缘、带标题栏），二选一。

### A3. AIRGAPPED 徽章叠在装饰框和左上提示条上
`:102-106` 徽章位于 `left-2 top-2`（还套在 `inset-2` 黄色装饰边框里），`:140` 的 "Workspace | Add devices…" 提示条在 `left-3 top-10`，与 `top-12` 中央条纵向只差 8px。三层左上角元素互不知道对方存在。截图里徽章直接压住后面的 tab 状元素。
**修**：左上角只保留一个复合状态条（AIRGAPPED 徽章并入其中），删装饰黄框或改为不占定位层。

### A4. Starter command 芯片截断
`page.tsx:905-919`：芯片 `truncate` + 容器横向滚动，第三条只剩 "Throw the red cube of…"，横向滚动条又贴着底部 catalog，可发现性差。**修**：芯片去 truncate 改多行 wrap（最多两行），或至少加省略提示与明显滚动指示。

### A5. 3D 内标注重叠
`SemanticDeviceStage.tsx:384/426/453`："LAYOUT SELECTED"、"Result Target"、"Expected Result" 都是世界坐标锚定的 HTML 标签，无碰撞避让，特定相机角度下两两重叠（截图中 "Expected Result/Result Target" 已叠成一团）。**修**：同锚点标签合并为一张卡；或加最简单的垂直错位（同屏投影距离 < 阈值时下移一行）。

## B. 多真相源（P0/P1）

### B1. 运行状态四套词表三个位置〔重构，可先做文案对齐=可修〕
- 顶栏：`Run / EXECUTED / Stop`（`page.tsx:2627-2637`，来源 `labReport.result`）
- AI 终端：`Run / Stop / Validate / COMPLETED + Execution completed.`（来源 `commandStatus`）
- Runtime Governor：`RUNTIME STATUS: COMPILED`（来源 `runtimeDecision`）
- 视口右上：`READY`（来源 `running` 布尔）

四者各自独立取数，语义确实不同（结果/命令流/编译决策/交互态），但界面上毫无区分说明，用户看到的就是"同一次运行，三个地方三个词"。
**短期〔可修〕**：统一色彩映射 + 各徽章加前缀（RUN:, CMD:, COMPILE:）。**长期〔重构〕**：一个全局 run-state store，单一状态条展示管线（compiled → simulated → executed），其余位置只订阅。

### B2. Run/Stop 按钮两套〔重构〕
顶栏和 AI 终端各一套，行为等价（都调 `runScenario`/`stopRun`）。保留一处（建议 AI 终端，因为输入在那里），顶栏只留状态。

### B3. "AIRGAPPED / SIMULATION ONLY" 单屏 4 次〔可修〕
视口徽章（`VirtualDeviceStage:104`，硬编码英文）、Runtime Governor 头（`AutonomyDecisionPanel:28/73`）、视口底部状态条尾部 `simulation_only`、AI 终端注意事项行。安全边界信息重要，但 4 次是稀释不是强调。**修**：保留视口徽章 + Governor 头两处，其余删。

### B4. 设备名 "Generic Industrial Robot Arm" 单屏 ≥8 次
左栏 Workspace Focus、中央条 ×2、右上覆盖层、视口底部状态条、AI 终端 active device 行、Governor TARGET DEVICE、Inspector 标题 + OVERVIEW 的 DEVICE/CURRENT RUN TARGET/MODEL/PROFILE 四行。
**短期〔可修〕**：Inspector OVERVIEW 去掉与标题重复的 DEVICE 行、与 PROFILE 相同值的 MODEL 行。**长期〔重构〕**：设备身份只在 Inspector 标题 + 视口选中标签两处。

### B5. 硬件状态三处〔重构〕
底栏 "REAL HARDWARE ■ not connected"、catalog 区 "REAL DISABLED" 按钮、右栏 RealHardwarePanel。真机验收后 REAL 路径会变重要，应收敛为右栏面板 + 底栏一个只读徽章（点击跳面板）。

### B6. "v0.1 选中即运行目标"解释出现 3 次〔可修〕
左栏 support_note、AI 终端蓝色行 `workspace_selection_run_same`、Inspector 里蓝色斜体行。留 AI 终端一处即可。

## C. 信息架构（P1，〔重构〕为主）

### C1. 左栏结构混乱
- 语言选择器塞在 "DEVICES" 标题行右侧（`LabConfigurator:100-107`）——语言是应用级设置，应去顶栏或设置菜单。
- "PUBLIC ALPHA SUPPORT" 整块（Supported Now / Current Selection / Workspace Focus）是**说明性内容**，却占左栏黄金位置且不可折叠。Supported Now 三个芯片是静态事实，Current Selection 与下拉框冗余，Workspace Focus 与视口/终端重复。整块可压成一行 support 徽章 + tooltip。
- Device Type / Profile / Scenario 三个下拉与"工作区设备"的关系没有视觉表达（改下拉会不会影响已摆放的设备?），首次用户无法建立模型。

### C2. AI 终端一个块里堆 10 类信息
`page.tsx:717-930`：输入框、Run/Stop/Validate、状态徽章、状态消息、LLM chip、active device 行、蓝色说明行、（不可运行时）双警告 + 黄色引导块、注意事项三连（| 分隔）、starter commands。垂直高度失控且每行字号都是 11px，无主次。
**重构方向**：输入 + Run/Stop + 单状态徽章为第一行；其余全部收进一个可展开的 "context" 折叠行。

### C3. 右栏 360px 三面板挤压
Governor 固定 42% 高、Inspector flex、RealHardwarePanel 底部（`page.tsx:2772-2807`）。EXECUTED 后用户最想看的审计时间线在 Inspector 深处，Governor 的 USER PROMPT/GOAL 却复述了用户刚输入的内容。**重构**：Governor 与 Inspector 改 tab 或手风琴；Governor 空闲态收起。

### C4. 顶栏按钮无分组语义
New/Open/Import Asset/Save/Save As/Restore/Quick Start 一排平铺，Restore 是为 conformance 测试保留的入口（代码注释自认），对用户语义不明。Validation Paths（截图中）与 Run 相邻但作用完全不同。**重构**：文件操作收进菜单（Electron 已有原生菜单栏），顶栏只留 Run 区 + 导出区。

## D. 文案与 i18n（P1/P2，多数〔可修〕）

- **D1〔可修〕** zh 模式下 Runtime Governor 字段名全是英文：`AutonomyDecisionPanel` 的 `copy.zh` 里 `userPrompt: 'User Prompt'`、`targetDevice: 'Target Device'` 等约 15 个键直接用英文值。要么补翻译，要么明确声明"面板术语保持英文"的设计决定并全局一致。
- **D2〔可修〕** 视口 AIRGAPPED 徽章硬编码英文（`VirtualDeviceStage:104`），zh 模式不翻译，与 Governor 头部同文案但一个走 i18n 一个不走。
- **D3〔可修〕** `localBlockedReason`（`VirtualDeviceStage:15-24`）靠英文关键词 includes 匹配翻译拦截原因，SafetyMonitor 新增 reason（本次 1.1/2.x 加了十几个）全部漏翻。应改为 reason code → i18n 键映射。
- **D4〔可修〕** Validate 按钮永久 `disabled`（`page.tsx:843-849`），无 tooltip 解释，是死 UI。加 "coming soon" title 或移除。
- **D5〔P2〕** `guidedPlaceholder` 里 `language === 'zh' ? '·' : '·'` 两个分支相同（死代码）；placeholder 把 starter 命令和提示拼在一起过长。
- **D6〔P2〕** "Expected Result / Result Target" 两个术语指同一概念的两半，视觉上却是两个独立标签；术语表应统一。

## E. 视觉一致性（P2，〔重构〕）

- **E1** 硬编码 hex 与设计 token 混用：`#232529`/`#2B2D31`/`#181A1D` 等散布 50+ 处，同时存在 `bg-bg-panel`/`border-border-panel` token 体系。改动主题时必然漏。
- **E2** 全界面 90% 文字是 `text-[11px]`，层级只靠 bold/uppercase/颜色微差，长时间使用扫描成本高。至少建立 11/13/15 三级并给面板标题升级。
- **E3** 视口覆盖层每个各配一套 `bg-black/24~45 + backdrop-blur`，透明度 6 种。统一为两级（信息层/操作层）。
